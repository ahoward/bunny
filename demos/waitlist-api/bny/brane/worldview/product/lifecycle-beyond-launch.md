# Lifecycle Beyond Launch
A waitlist is a point-in-time tool — what happens after launch day, and how does the API avoid becoming disposable?

## The Disposability Problem
Waitlists are inherently temporary. Once the product launches, the waitlist drains and the API is abandoned. This limits LTV and creates churn by design.

## Post-Launch Extensions
- **Beta cohort management** — promote users in waves, track activation per cohort
- **Feature waitlists** — queue users for upcoming features (recurring use case)
- **Capacity management** — throttle signups during scaling, re-queue overflow
- **Re-engagement campaigns** — win back users who signed up but never activated

## The Platform Play
If the API captures the relationship between a founder and their early users, it can evolve into:
- Early user CRM
- Launch analytics (conversion from signup → activation)
- Community seeding tools

## Risk
Scope creep destroys focus. The MVP should nail the waitlist use case completely before expanding. But the architecture should not preclude these extensions.
