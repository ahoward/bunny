## what was built

### models
- `Event` -- title, description, location, start/end times, capacity, accepted_count, time_zone
- `User` -- name, email
- `Rsvp` -- event, user, status, party_size, waitlist_position, undo_token, undo_expires_at
- `RsvpTransition` -- audit trail of state changes

### controllers
- `EventsController` -- CRUD with cursor pagination
- `RsvpsController` -- create (auto-accept/waitlist), cancel, undo, confirm promotion

### services
- `RsvpCreator` -- pessimistic locking, capacity check, auto-waitlist
- `WaitlistPromoter` -- FIFO promotion, group skip-and-retain, idempotent
- `RsvpCanceller` -- decrement headcount, trigger promotion
- `RsvpUndoer` -- token-based undo within grace window

### jobs
- `WaitlistPromotionJob` -- async promotion after cancellation
- `UndoExpiryJob` -- expires undo tokens after grace period

### migrations
- `create_users` -- name, email, timestamps
- `create_events` -- full event schema with check constraints
- `create_rsvps` -- with unique constraint on (user_id, event_id)
- `create_rsvp_transitions` -- audit trail

### tests (written by gemini, code by claude)
- `tests/contract/events_api_spec.rb` -- API contract tests
- `tests/property/rsvp_capacity_spec.rb` -- capacity invariant properties
- `tests/boundary/edge_cases_spec.rb` -- edge cases from challenge
- `tests/golden/api_responses_spec.rb` -- golden file response snapshots
