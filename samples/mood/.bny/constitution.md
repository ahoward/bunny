# Constitution

## Principles

1. **POD only** — plain old data, no classes for data containers
2. **Guard early** — return errors at function top
3. **snake_case** — variables, functions, file names
4. **null over undefined** — explicit absence
5. **Simplicity** — three similar lines > one premature abstraction
6. **Tests first** — design tests, then implement

## Naming

| Thing | Style | Example |
|-------|-------|---------|
| Constants | SCREAMING_SNAKE | `MAX_SCORE` |
| Types | PascalCase | `MoodEntry` |
| Variables/functions | snake_case | `get_moods` |

## Terminology

- `params` for input
- `result` for output
